БЭМ (Блок, Элемент, Модификатор)

БЭМ — это методология, разработанная Яндексом, которая помогает создавать переиспользуемые компоненты и упрощает совместную разработку. Она использует строгую нотацию именования: блок__элемент--модификатор, что обеспечивает четкую структуру и предотвращает конфликты стилей.

  • Блок: Независимый компонент, например, header или button.
  • Элемент: Часть блока, которая не может использоваться отдельно, например, header__logo.
  • Модификатор: Вариация стиля или поведения, например, button--primary.

Преимущества БЭМ

БЭМ обеспечивает строгую организацию кода, что делает его идеальным для крупных проектов. Основные преимущества включают:

  • Четкая структура благодаря строгой нотации.
  • Легкость переиспользования компонентов в разных проектах.
  • Уменьшение конфликтов стилей за счет уникальных имен классов.
  • Упрощение поддержки и масштабирования кода.

Пример реализации

Вот пример кнопки, реализованной с использованием БЭМ: 

<button class="button button--primary">Нажми меня</button>

<style>
.button {
    display: inline-block;
    padding: 10px 20px;
    font-size: 16px;
    border-radius: 4px;
}
.button--primary {
    background-color: var(--secondary-color);
    color: white;
    border: none;
}
.button--primary:hover {
    background-color: #2980b9;
}
</style>

Этот пример демонстрирует, как блок button и его модификатор button--primary создают переиспользуемый компонент с четкой структурой.
