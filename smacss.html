SMACSS (Масштабируемая и модульная архитектура для CSS)

SMACSS, разработанная Джонатаном Снуком, — это методология, которая классифицирует стили на пять категорий: базовые, макет, модули, состояния и темы. Это помогает структурировать CSS-код и упрощает его масштабирование в больших проектах.

  • Базовые: Основные стили, такие как сбросы или типографика (например, body, h1).
  • Макет: Правила для компоновки страницы, например, сетки или контейнеры.
  • Модули: Переиспользуемые компоненты, такие как кнопки или карточки.
  • Состояния: Стили для состояний, например, .is-active или :hover.
  • Темы: Визуальные стили для изменения внешнего вида (например, темная или светлая тема).

Преимущества SMACSS

SMACSS особенно полезна для крупных проектов, где требуется четкая организация. Основные преимущества:

  • Логичная категоризация стилей для удобной навигации.
  • Поддержка масштабируемости за счет модульной структуры.
  • Упрощение управления состояниями и темами.
  • Гибкость при адаптации к различным проектам.

Пример реализации

Вот пример кнопки, реализованной с использованием SMACSS: 

<button class="module-button is-primary">Нажми меня</button>

<style>
.module-button {
    display: inline-block;
    padding: 10px 20px;
    font-size: 16px;
    border-radius: 4px;
}
.is-primary {
    background-color: var(--secondary-color);
    color: white;
    border: none;
}
.is-primary:hover {
    background-color: #2980b9;
}
</style>

В этом примере класс module-button определяет модуль, а is-primary задает состояние, что соответствует принципам категоризации SMACSS.
